Congestion window under loss

Senders infer network capacity from ACK timing and loss signals. Loss often shrinks the congestion window (cwnd); RTT caps how fast in-flight data can drain the pipe even when cwnd is large.

BDP

Bandwidth-delay product ≈ rate × RTT. To fill a fat long-haul link you need enough bytes in flight—window scaling, buffer sizing, and application parallelism all interact.

Operator hint

Tail latency spikes often correlate with loss or bufferbloat on a path segment—measure with RTT variance and retransmit counters, not only averages.

Toy: cwnd vs loss

Real congestion control (Cubic, BBR, …) is stateful and path-specific. The site explorer uses a deliberately simple coupling: raising loss causes visible retransmit events in the packet stream and drags down a toy cwnd trace; completed round trips let it creep up again. Use it to build intuition, not to predict production throughput.
