DNS and TLS latency

Before application bytes flow, the client usually pays for name resolution, TCP setup, and TLS handshakes. On short requests, that setup dominates time-to-first-byte.

DNS
  • Resolver cache hits are fast; cold lookups traverse NS chains and may add multiple RTTs.
  • Happy Eyeballs and IPv6/IPv4 fallbacks can add parallel connection attempts—good for reliability, noisy for benchmarks.
TLS
  • 1-RTT vs 0-RTT resumption trades latency for careful replay policy.
  • Certificate validation and OCSP stapling quality affect tail latency more than median.
